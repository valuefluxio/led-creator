The Foundation

"Cache Rules Everything
Around Me."

  • Prompt caching lets you reuse computation from previous interactions
  • Dramatically reduces both latency and cost in long-running agents
  • Works via prefix matching — the API caches everything from the request start to each cache_control breakpoint

How It Works

Prefix
Matching

The cache stores everything from the start of the request to each cache_control breakpoint.

The Critical Rule:

Any change anywhere in the prefix invalidates everything after it.

Design your entire system around this constraint — get the ordering right and most caching works for free.

The Golden Principle

Static First.
Dynamic Last.

① System prompt + tools globally cached
② CLAUDE.md documentation cached per project
③ Session context cached per session
④ Conversation messages dynamic — changes every turn

Lesson 01

Use messages,
not system prompt changes.

DON'T

Edit the system prompt to inject plan mode, timestamps, or file changes — this invalidates the entire cached prefix.

DO

Insert updated context via <system-reminder> tags in the next user message or tool result instead.

Lesson 02

Model switching
is more expensive
than you think.

  • Prompt caches are model-specific — switching models destroys the cache
  • 100k tokens deep in Opus? Switching to Haiku costs more — you rebuild the entire cache
  • Solution: use subagents where the primary model hands off specific tasks
06 / 11 trq212 · Anthropic

Lesson 03

Never add or remove
tools mid-conversation.

Plan Mode

Use EnterPlanMode / ExitPlanMode as callable tools — not by swapping the tool set.

Tool Search

Deploy lightweight stubs with defer_loading flags instead of removing tools. Let the model discover full schemas when needed.

Lesson 04

Cache-safe
context forking. 

# When context window fills up — don't restart.
# Fork cache-safely instead:

new_conversation = fork({
  system_prompt:  identical,   # reuses cache
  tools:          identical,   # reuses cache
  messages:       parent_msgs, # prepend full history
  append:         compaction_prompt # new user msg
})

# Result: only NEW tokens are charged.
# The cached prefix is reused entirely.

Operations

Treat cache hit rate
like uptime.

  • Claude Code runs alerts on prompt cache hit rates — it's a first-class metric
  • The team declares SEVs (severity incidents) when rates drop too low
  • Even small percentage drops cause dramatic increases in cost and latency
  • High cache hit rates = lower costs → more generous rate limits for users
09 / 11 trq212 · Anthropic

Watch Out For

Cache killers.

01

Timestamps in static prompts — the date changes every day, busting the cache every session

02

Non-deterministic tool ordering — tools must be in the same order every request

03

Switching models to save cost — rebuilding a 100k token cache often costs more than staying on the original model
